        本说明详细介绍汉英机译系统所使用的各类数据文件格式。
        在本说明中，括在尖括号中的部分，如 <Ident>，表示 Ident 是本说明中用到的概
    念。而尖括号中如果出现冒号，如 <Ident:LexCat>，则表示 LexCat 是对概念 Ident
    的一些限定。
        以下例子介绍本文在说明一个概念时所采用的形式：
        <PhrAtt>:
                 PhrAtt : { <Att_Set> }
                 PhrAtt <Condition> : { <Att_Set> }
        在这个例子中，第一行表示被说明的概念是<PhrAtt>，后面两行表示这个概念有两
    种形式。后面两行中又用到了关键字 PhrAtt, PhrCat, 分隔符 ( ) { } : , 和其它几
    个概念：<Attribute>, <Att_Set>, 和 <Condition>。

§1 语言模型

        语言模型文件（缺省文件名为MODEL）定义本系统中所有的描述用的符号。包括词、
    短语、句子的分类及属性，词的语法语义关联等。它包括以下几项内容：
        <Model>:
                 <SemModel>
                 <Model> <SynModel>
        <SemModel>:
                 SemModel <SemModelBody> EndModel
        <SemModelBody>:
                 <SemCat> <SemFeat> <SemAss>
        <SynModel>:
                 SynModel ( <LangCode> ) <SynModelBody> EndModel
        <SynModelBody>:
                 <LexCat> <LexFeat> <PhrCat> <PhrFeat> <SynAss> <SpecialField>
        其中SemModel和EndModel之间的是语义模型，SynModel和EndModel之间是句法模型。

§1.1 语义分类

        <SemCat>:
                 SemCat : <Attribute> : { <HieSym_Set> }
        语义分类是一个多级分类系统。
        语义分类属性的类型为符号型，活跃模式为静态，相关模式为不相关。
        在特征结构描述中，被说明为 SemCat 的属性之值应为 <Ident:SemCat>。

        <SemAtt> 是属性说明，用于列出所有的语义属性：
                SemCat 类的属性只有一个，用于表示语义分类，其类型为层次型，取值范围
                为前面<SemCat>中所说明的语义分类；
                <Condition>后列出该条件成立时所具有的属性。
        <SemVal> 是属性值说明，说明语义属性的类型、范围、缺省值和模式。
        <AtomVal> 是原子属性的取值说明。
        <AtomType>用于说明属性的取值类型和范围：
                Hierar 为层次型；
                Symbol 为符号型；
                Number 为数值型；
                Boolean 为布尔型；
                Limited 说明取值范围为有限集合；
                Unlimited 说明取值范围为无限集合，即可取该类型的任意值；
        <HieSym_Set> 为有限的层次类型和符号类型指定取值范围，
                并可为单个层次型值或符号型值定义若干个别名；
        <Number_Set> 为某些数值定义若干个别名；
        <Boolean_Set> 为布尔值定义若干个别名。
                有关各种类型的原子的定义，参见后面§2 节。
        <DefAtom> 用于说明属性的缺省值(一个原子)，该值应符合前面的说明。
        <ValMode> 用于属性的模式说明。
        <ActMode> 用于说明属性的活跃模式。
                属性的活跃模式可以是静态的(Static)或动态的(Dynamic)。
        <RelMode> 用于说明属性的相关模式。
                相关模式有两种：RelMode=0 表示不相关，RelMode=1 表示相关。
                属性的相关模式反映了一个语言成分的该属性之值与其中心语言成分的该属性之值
                之间的关系。
        语义属性的缺省活跃模式为静态，缺省相关模式为相关。

§1.3 语义关联

        <SemAss>:
                 <SemAssAtt>
                 <SemAssVal>
                 <SemAss> <SemAss>
        <SemAssAtt>:
                 SemAssAtt : { <Att_Set> }
                 SemAssAtt <Condition> : { <Att_Set> }
        <SemAssVal>:
                 SemAssVal ( <Ident:SemAssAtt> ) : <FestVal>
        <FestVal>:
                 <DefFest>
                 <ValMode>
                 <DefFest> <ValMode>
        <DefFest>:
                 Default = <Fest>
        关联属性与其它属性不同的是，它的取值不是原子，而是另外一个特征结构。
        语义关联用于建立句子中语言成分之间的语义关系，用于表示语言成分的深层格。
        <SemAssAtt> 是语义关联属性说明，用于列出所有的语义关联属性：
        <SemAssVal> 是语义关联属性值说明，说明语义关联属性的缺省值和模式。
        <FestVal> 是特征结构属性的取值说明。
        <DefFest> 用于说明属性的缺省值(一个特征结构)。
        语义关联属性的缺省活跃模式为静态，缺省相关模式为相关。

§1.4 词语分类

        <LexCat>:
                 LexCat : <Attribute> : { <HieSym_Set> }
        这一部分用于说明一种语言的词语的分类系统。该分类系统是一个枚举的符号类型。
        <Attribute> 说明词类属性的属性名，<HieSym_Set> 说明具体有哪些词类及子类。
        词语分类属性的类型为符号型，活跃模式为静态，相关模式为相关。
        在特征结构描述中，被说明为 LexCat 的属性之值应为<Ident:LexCat>。

§1.5 词语的语法特征

        <LexFeat>:
                 <LexAtt>
                 <LexVal>
                 <LexFeat> <LexFeat>
        <LexAtt>:
                 LexAtt ( BaseForm ) : <Attribute>
                 LexAtt ( VaryForm ) : <Attribute>
                 LexAtt : { <Att_Set> }
                 LexAtt <Condition> : { <Att_Set> }
        <LexVal>:
                 LexVal ( <Ident:LexAtt> ) : <AtomVal>
        这一部分描述词语的那些仅靠分类所无法描述的语法特征。
        <LexAtt>是属性说明，用于列出所有的词类语法属性：
                LexBase 所指明的是单词原形属性，用于表示单词的原形，其类型为符号型，
                        活跃模式为静态，相关模式为相关；
                LexVary 所指明的是单词变形属性，用于表示单词的实际形态（如经过屈折或
                        重叠后），其类型也是符号型，活跃模式为动态，相关模式为相关；
                <Condtion>是一个表示条件的特征结构，后面列出只有当该条件成立时才具有
                        的属性，其类型、取值范围、缺省值和模式在后面<LexVal>中说明；
                Static类的属性是静态属性；
                Dynamic类的属性是动态属性。
        <LexVal>是属性值的说明，说明语法属性的取值类型、范围、缺省值和模式，
                应对每一属性分别说明。
        词语属性的缺省活跃模式为静态，缺省相关模式为相关。

§1.6 短语句子分类

        <PhrCat>:
                 PhrCat : <Attribute> : { <HieSym_Set> }
        这一部分用于说明短语和句子的分类系统，它也是一个枚举的符号类型。其说明格式
        类似于词语分类的说明。
        词语的分类和短语句子的分类不能同名，它们一起用作语言成分的标记(Label)。
        词语的分类系统以及短语句子的分类系统组成了一部短语结构语法(PSG)。
        短语句子分类属性的类型为符号型，活跃模式为静态，相关模式为不相关。
        在特征结构描述中，被说明为PhrCat的属性之值应符合<PhrCat>类型。

§1.7 短语句子的语法特征

        短语句子属性的缺省活跃模式为静态，缺省相关模式为不相关。

§2 描述语言知识的基本形式

        本节介绍本系统中所用的几种描述语言知识(包括规则、词典和短语、句子结构等)的
    基本形式。

§2.1 原子

        原子是最基本的描述单位，它不可以再分割，但可以用‘与’，‘或’，‘非’等逻
    辑符号相联结。原子之间的基本运算是合一。
        <Atom>:
                 <Hierar>
                 <Symbol>
                 <Number>
                 <Boolean>
        <Hierar>:
                 <Hierar_Or>
        <Hierar_Or>:
                 <Hierar_And>
                 <Hierar_Not>
                 <Hierar_Or>|<Hierar_And>
        <Hierar_And>:
                 <Hierar_Smp>
                 <Hierar_And>~<Hierar_Smp>
        <Hierar_Not>:
                 ~<Hierar_Smp>
                 <Hierar_Not>~<Hierar_Smp>
        <Hierar_Smp>:
                 <Ident:Atom_Name>
                 <Ident:Atom_Alias>
        <Symbol>:
                 <Symbol_Or>
                 <Symbol_Not>
        <Symbol_Or>:
                 <Symbol_Smp>
                 <Symbol_Or>|<Symbol_Smp>
        <Symbol_Not>:
                 ~<Symbol_Smp>
                 <Symbol_Not>~<Symbol_Smp>
        <Symbol_Smp>:
                 <Ident:Atom_Name>
                 <Ident:Atom_Alias>
        <Number>:
                 <Number_Or>
                 <Number_Not>
        <Number_Or>:
                 <Number_Smp>
                 <Number_Or>|<Number_Smp>
        <Number_Not>:
                 ~<Number_Smp>
                 <Number_Not>~<Number_Smp>
        <Number_Smp>:
                 <Integ>
                 <Ident:Atom_Alias>
        <Boolean>:
                 <True>
                 <False>
                 <Ident:Atom_Alias>
        原子<Atom>主要有四种不同的形式：层次型、符号型、数值型和布尔型。进行运算时，
    所有的别名 <Atom_Alias> 都要转换成其基本形式 <Atom_Name>,<Integ>,<True>,<False>。
        布尔型<Boolean>最简单，它只有<True>和<False>两个值，它不能组成与、或、非等
    复合形式，布尔值只能与它本身合一。
        数值型<Number>可以取所有自然数或零，单个数值只能与其本身合一。数值可以组成
    两种复合形式：~m~n...(非m且非n...)和m|n|t...(m或n或t...)。
        符号型<Symbol>可以取标识符为值，单个符号只能与其本身合一。其组合形式与数值
    型相同。
        层次型<Hierar>比较复杂。假定有单个层次型值α和单个层次型值β，当且仅当存在
    一个串γ，能满足α＝βγ（或β＝αγ）时，α和β才能够合一，合一的结果为α（或
    β）。这时我们称α为β（或β为α）的下位值，β为α（或α为β）的上位值。它们反
    映了多级分类系统中上位范畴和下位范畴之间的关系。 层次型有三种复合形式：~A~B...
    (非A且非B...)、A~A1~A2...(A且非A1且非A2...) 以及这两种形式之或(|)。 对于第二种
    形式，要求A1, A2等必须是A的下位值。

§2.2 特征结构

        特征结构用于语言单位（如词语、短语、句子等）的内部结构，也用于描述规则和词
    典。它的基本形式是“属性--值”对的集合。
        <Fest>:
                 [ ]
                 { }
                 [ ] { }
                 [ <IntFest> ]
                 { <ExtFest> }
                 [ <IntFest> ] { <ExtFest> }
                 <None>
        <IntFest>:
                 <IntFeat>
                 <IntFest> , <IntFeat>
                 <IntFest> ; <IntFest>
        <ExtFest>:
                 <ExtFeat>
                 <ExtFest> , <ExtFeat>
        <IntFeat>:
                 <IntAtt> : <IntVal>
        <ExtFeat>:
                 <ExtAtt> : <ExtVal>
        <IntAtt>:
                 <Ident:LexAtt>
                 <Ident:PhrAtt>
                 <Ident:SemAtt>
        <ExtAtt>:
                 <Ident:SemAss>
        <IntVal>:
                 <Atom>
        <ExtVal>:
                 <Fest>
        <None>:
                 无
                 None
                 NONE
        特征结构<Fest>由一些特征组成，每个特征是一个属性--值对。
        值为原子的特征，称为内部特征<IntFeat> ，所有内部特征括在方括号中，组成内
    部特征集合<IntFest>。 在方括号中，逗号表示"与"，分号表示"或"，与的优先级高
    于或的优先级。内部属性<IntAtt>是一个表示属性的标识符。内部值<IntVal>是一个原
    子。
        值为其它特征结构的特征，称为外部特征<ExtFeat>，所有外部特征括在花括号中，
    组成外部特征集合<ExtFest>。外部特征集中不允许出现表示"或"的分号。外部属性
    <ExtAtt>是一个表示关联的标识符。外部值<ExtVal>是另外一个特征结构。
        <None>是一个特殊的特征结构, 除了它自身以外, 它不能与任何其它特征结构合一。

§2.3 树和森林

        <LeftForest>:
                 <LeftTree>
                 <HeadTag> <LeftTree>
                 <LeftForest> <LeftForest>
        <LeftTree>:
                 <LabelItem>
                 <LabelItem> < <WordItem_Or> >

                 <TransVarTag> <LabelItem>
        <LabelVarTag>:
                 %
                 %%
                 %%%
                 %%%%
        <TransVarTag>:
                 /
                 //
                 ///
                 ////
        树与树林是语言知识表示中一种常用的形式，用于表示语言成分的内部结构。树的结
    点对应于句子中的语言成分。
        在使用中，树与树林有两种形式，一种是左树<LeftTree>与左树林<LeftForest>，用
    于与某一语言成分的内部结构相匹配，一种是右树<RightTree>与右树林<RightForest>，
    用于表示某一棵左树经过转换后得到的结果。左树和右树总是成对出现的，其结构基本类
    似，个别地方有所不同，以下详细描述。
        树林<LeftForest>和<RightForest>由若干个带中心标志的树组成。
        中心标志<HeadTag>表示该结点是父结点的中心子结点。
        树<LeftTree>和<RightTree>由一个标记项加上一个可选的单词项，或加上一个可选的
    子树林组成；标记项是一个表示词类或短语句子类的标识符；单词项由若干个单词之"或"
    组成，其中~用于词典中，表示词典中当前词记录的入口单词。
        子树林<LeftSubForest>和<RightSubForest>由若干棵加中心标志的子树组成，对于
    右子树林而言，还可以包括一些加复制标志、移动标志或删除标志的位移标记项。
        子树<LeftSubTree>和<RightSubTree>与树类似，只是对于右子树而言，标记项后面可
    加一个转换变量，表示该子树是由相应左树中对应于转换变量的子树转换而来，如果没有
    转换变量，则表示该子树是新增加的。
        位移标记项<ShiftLabelItem>由一个标记项加上一个标记变量组成，表示该位移操作
    的源结点是树中对应于该标记变量的子树的某一个深层子树。
        标记变量由ｎ个百分号加上一个标记项组成，表示对应树中的第ｎ个该标记项的结点。
        转换变量由ｎ个斜杠号加上一个标记项组成，表示右树的对应左树中的第ｎ个该标记
    项的结点。（以上ｎ= 1,2,3,4）
    关于树结构，还要注意以下几点：
    ● 关于树所对应的语言：
       1．一棵树对应唯一的一种语言；
       2．除右树中的转换变量以外，该树中所有的标记都是这种语言的标记；
       3．右树中的转换变量中的标记，是对应左树的语言的标记；
       4．左树和右树可以对应不同种语言（如在析句规则中），
          也可以对应同一种语言（如在造句规则中）；
    ● 关于中心子结点：
       1．一个父结点可以没有中心子结点，或者有一个中心子结点；
       2．父结点和中心子结点共享该树所对应语言的相关属性(即 RelMode=1 的属性)，
          但这种共享关系可以被切断（参见约束<Bind>）；
       3．一个标记项前面如果有中心标志，那么它所对应的结点必须是其父结点的中心子结点；
       4．一个标记项前面如果没有中心标志，那么它所对应的结点必须不是其父结点的中心
          子结点；
    ● 在左树到右树的转换中：
       1．左树的一个结点最多只能转换到右树的一个结点；
       2．根结点必须转换到根结点；
       3．叶结点必须转换到右树的叶结点, 或者在右树中不出现；
       4．中间结点必须转换到右树的中间结点, 或者在右树中不出现；

    ● 在左树到右树的转换中，如果左树和右树对应同一种语言，那么：
       1．如果一个结点是另一个结点的中心子结点，那么转换后该结点必须仍然是其中心项
          子结点；
       2．如果一个结点不是另一个结点的中心子结点，那么转换后该结点必须仍然不是其中
          心子结点；
       3．如果一个结点没有中心子结点，那么转换后可用新产生的结点作为其中心子结点；
    ● 关于位移结点：
       1．在右树中，位移标记项对应的结点称为位移结点，一个位移结点对应一次位移操作;
       2．位移操作包括复制、移动和删除三钟操作，位移操作在整棵树的转换结束后进行；
       3．位移操作的目的结点就是位移结点，目的结点用于指明位移操作的目的位置和被实
          施位移操作的结点所应满足的特征结构；
       4．位移操作的源结点是位移标记项中的标记变量所对应的结点的某一个深层子结点，
          源结点的特征结构和目的结点的特征结构必须能够合一；
       5．复制操作保留源结点，并复制一份到目的结点位置；
       6．移动操作将源结点移动到目的结点位置；
       7．删除操作只删除源结点，目的结点位置不产生任何变化；
       8．位移操作的源结点中不能含有另一次位移操作的源结点或目的结点作为其子孙结点,
          即位移操作不能嵌套（避免出现递归位移）；
       9．可以对复制操作的源结点和目的结点以及位移操作的目的结点再次执行位移操作。

§2.4 约束

        <Bind>:

                 IF <Bind> THEN <Bind> ELSE <Bind> ENDIF
        <Equation>:
                 <IntVar> = <Atom>
                 <IntVar> = <IntVar>
                 <ExtVar> = <Fest>
                 <ExtVar> = <ExtVar>
                 <ExtVar> = <True>
                 <ExtVar> = <False>

                 <ExtVar> != <ExtVar> <At>

                 <ExtVar>.<IntAtt>

        约束用于对一棵树中各个结点的特征结构进行限制；
        约束<Bind>由多个条件约束表达式和约束等式组成；
        条件约束表达式<Test>用于指明在一定条件下才需要进行的约束；
        约束等式<Equation>是一个表示合一运算的等式，这种等式有四类：
        1．一个内部变量与一个原子或另一个内部变量合一；
        2．一个外部变量与一个特征结构或另一个外部变量合一；
        3．一个外部变量与一个布尔值合一，用于判断该变量是否已指向句中的语言成分；
        4．强制两个外部变量共享（==）或不共享（!=）某些属性。

§3 析句规则

        缺省的析句规则文件名为 PRSRBASE ( Sentence-Parsing RuleBase )。
        <PrsRule>:
                 && { <Title> } <PrsStruct> || <TrnSwitch> => <TrnStruct>
                 && { <Title> } <PrsStruct> => <TrnStruct>
                 <PrsRule> || <TrnSwitch> => <TrnStruct>
                 <PrsRule> => <TrnStruct>
        <Title>:
                 <Ident>
                 <Ident> / <Note>
        <Note>:
                 <Ident>
                 <Note> <Ident>
                 <Note> , <Ident>
        <PrsStruct>:
                 <PrsReductn> -> <PrsForest>
                 <PrsReductn> -> <PrsForest> :: <PrsBind>
        <PrsReductn>:
                 <LabelItem>
        <PrsForest>:
                 <LeftForest>
                 ^ <LeftForest>
        <PrsBind>:
                 <Bind>
        <TrnSwitch>:
                 <Bind>
        <TrnStruct>:
                 <TrnTree>
                 <TrnTree> <TrnBind>
        <TrnTree>:
                 <RightTree>
        <TrnBind>:
                 <Bind>
        析句规则<PrsRule>由一个析句结构与若干个转换开关和转换结构组成。
        析句规则名<Title>是一个标识符, 它在整个规则库中应唯一对应一条规则。
        析句结构<PrsStruct> 由一个析句归约、一个析句树林与一个可选的析句约束组成。
    析句结构实际上也是一棵树，其根结点是析句归约，内部结构是析句树林，析句约束表
    示的是对析句结构这棵树（而不是对析句树林中的某棵树）的约束。
        析句归约<PrsReductn>是一个标记项。
        析句树林<PrsForest>由树林与可选的左右边界(^)组成。
        转换开关<TrnSwitch>实际上是个对源语言树的约束，每个转换开关后接一个或多个
    转换结构，如果转换开关中的约束不被满足，则后面的转换结构都不会被使用，直到下一
    个转换开关。转换开关可以省略，表示没有约束。
        转换结构<TrnStruct>由一棵转换树与一个可选的转换约束组成。
        析句树<PrsTree>是左树。
        转换树<TrnTree>是右树。
        析句约束<PrsBind>和转换约束<TrnBind>分别是对源文和译文的约束。

        缺省的造句规则文件名为 MAKRBASE ( Sentence-Making RuleBase )。

                 <LMakTree>

        <LMakBind>:

                 <RMakTree>

        <RMakTree>:
                 <RightTree>
        <RMakBind>:
                 <Bind>
        造句规则<MakRule>由一个左造句结构和一到多个右造句结构组成。

        左造句约束<LMakBind>和右造句约束<RMakBind>都是约束。

§5 词典

        缺省的词典文件名DICT。
        <Entry>:
                 $$ <Word> <Meaning>
                 <Entry> <Meaning>
                 <Entry> <PrsRule>
        <Meaning>:
                 ** { <Title> } <SrcMeaning>
                 <Meaning> || <TrnSwitch> => <TgtMeaning>
                 <Meaning> => <TgtMeaning>
        <SrcMeaning>:
                 <LabelItem>
                 <LabelItem> <Bind>
        <TgtMeaning>:
                 <TrnStruct>
        词典是机器翻译所需的语言知识的主要承载体。
        词典中一个词记录<Entry>由单词名、若干义项和若干局部析句规则。
        义项<Meaning>由一个义项名，一个源文义项与若干个转换开关和译文义项组成。
        义项名<Title>是一个标识符, 它在一个词记录中对应唯一的一个义项； 同样, 一个
    词记录中的每一条局部析句规则<PrsRule>也有一个唯一的规则名<Title>。
        源文义项<SrcMeaning>由一个源文标记项（词类标记或短语句子分类标记）和一个可
    选的特征结构组成，特征结构中包含了该词在源文分析时所需的各类信息，包括各类语法
    语义特征以及语义关联。
        译文义项<TgtMeaning>与析句规则中的转换结构相同，包括一棵译文树和一个译文约
    束。译文树是一棵右树。
        词典中的局部析句规则提供该词专用的，在析句规则库中所没有的一些规则，包括各
    种成语及习惯用法。

§6 汉语重叠词规则

        缺省的文件名为 OVLRBASE ( Word-overlapping RuleBase )。
        <OvlRule>:
                 @@ <OvlLexVaryStruct> << <OvlLexBaseStruct>
        <OvlLexVaryStruct>:
                 <OvlLexVary> -- <OvlLexStruct>
        <OvlLexBaseStruct>:
                 <OvlLexBase> -- <OvlLexStruct>
        <OvlLexStruct>:
                 <LabelItem>
                 <LabelItem> <Fest>
        <OvlLexVary>:
                 <ChnChar>
                 <OvlCharVar>
                 <OvlLexVary> <OvlLexVary>
        <OvlCharVar>:

        <OvlLexBase>:
                 <ChnChar>
                 <OvlCharVar>
